    Tóm tắt nội dung

    3. Tóm tắt nội dung:

    Đề tài tập trung xây dựng một hệ thống sàn thương mại điện tử chuyên biệt cho lĩnh vực bất động sản, hoạt động trên cả hai nền tảng Web và Mobile App. Hệ thống được thiết kế theo kiến trúc Microservice, chia nhỏ các nghiệp vụ thành các dịch vụ độc lập để dễ dàng mở rộng và bảo trì, được đóng gói và triển khai tự động hóa bằng Docker.

    Hệ thống hoạt động như một sàn giao dịch trung gian: Người bán (Môi giới/Chủ nhà) có thể nạp tiền vào ví điện tử để trả phí đăng tin hoặc mua các gói dịch vụ ưu tiên hiển thị (Tin VIP, đẩy tin). Người mua có thể tìm kiếm bất động sản thông minh theo vị trí, khoảng giá và các tiện ích xung quanh. Dữ liệu hành vi người dùng được thu thập để phân tích và gợi ý các bất động sản phù hợp nhất. Ngoài ra, hệ thống tích hợp Chatbot AI để hỗ trợ tư vấn sơ bộ và giải đáp thắc mắc tự động.

    ➤ Các công nghệ được sử dụng :

    - Frontend: ReactJS (Web), React Native (Mobile App)

    - Backend: Java Spring Boot

    ➤ Kiến trúc: Microservice

    - Cơ sở dữ liệu: PostgreSQL

    - Giao tiếp: REST API, API Gateway

    - Bảo mật: JWT

    - Container: Docker, Docker Compose

    ➤ AI hỗ trợ: Chatbot API

    ➤ Công cụ: Jira/Trello, Git, Postman, Agile/Scrum, CI/CD (Jenkin)

    ➤ Hạ tầng triển khai: AWS (EC2, RDS, API Gateway – mô phỏng)

    ➤ Hệ thống tổng quát:

    Hệ thống bao gồm các Microservice chính:

    – Identity Service: Quản lý người dùng, xác thực (Auth), phân quyền (Admin, Môi giới, Người tìm).

    – Property Service: Quản lý tin đăng, danh mục BĐS, xử lý hình ảnh/video.

    – Payment & Wallet Service: Quản lý ví tiền, lịch sử giao dịch, tích hợp cổng thanh toán (Giả lập hoặc thực tế).

    – Search & Recommendation Service: Tìm kiếm nâng cao, gợi ý tin đăng.

    – Chatbot & Support Service: Xử lý hội thoại tự động.

    → Frontend Web và Mobile App giao tiếp với hệ thống qua API Gateway

    ➤ Phân công công việc:

    SINH VIÊN 1: THỰC HIỆN LẬP TRÌNH FRONTEND WEB (REACTJS):

    – Xây dựng giao diện Web cho hệ thống

    – Xây dựng trang quản trị (Admin Dashboard): Quản lý người dùng, duyệt tin đăng, xem báo cáo doanh thu,chat realtime.

    – Xây dựng trang cá nhân cho Môi giới: Quản lý tin đã đăng, thao tác nạp tiền, xem lịch sử trừ tiền,chat realtime.

    – Xây dựng trang chủ: Các thông tin về bất động sản, trang filter tìm lọc, Map bản đồ khu vực,chat realtime,Chatbox Ai,gợi ý đề xuất

    – Gọi và tối ưu UI/UX hiển thị dữ liệu từ các REST API của hệ thống Microservice

    SINH VIÊN 2: THỰC HIỆN LẬP TRÌNH MOBILE APP

    – Xây dựng ứng dụng Mobile cho hệ thống

    – Xây dựng chức năng tìm kiếm BĐS dựa trên vị trí hiện tại (sử dụng GPS và Bản đồ)

    – Chức năng nhắn tin, đăng tin nhanh trên điện thoại (chụp ảnh, upload).

    – Nhận thông báo (Notification) khi có khách hàng liên hệ hoặc khi tin đăng sắp hết hạn.

    – Đồng bộ dữ liệu tài khoản và ví với nền tảng Web

    – Chatbox AI

    SINH VIÊN 3: THỰC HIỆN LẬP TRÌNH – BACKEND (THỐNG KÊ, CÁ NHÂN HÓA & AI)

    – Xây dựng Identity Service: Đăng ký, đăng nhập, JWT, phân quyền.

    – Xây dựng Property Service: CRUD tin đăng, quản lý địa chính (Tỉnh/Huyện/Xã), upload file

    – Thiết lập cấu hình Docker & Docker Compose cho toàn bộ nhóm để chạy hệ thống.

    – Thiết kế CSDL

    – Đóng gói service bằng Docker

    SINH VIÊN 4: THỰC HIỆN LẬP TRÌNH – BACKEND (FINTECH, ANALYTICS & AI)

    – Xây dựng Payment & Wallet Service: Logic ví điện tử (cộng/trừ tiền), quản lý bảng giá dịch vụ (Tin thường/Tin VIP).

    – Xây dựng Analytics Service: Thống kê số lượt xem, báo cáo doanh thu hệ thống.

    – Tích hợp Chatbot API và API gợi ý bất động sản.

    – Chat Realtime (Websocket), Notification

    – Đóng gói và triển khai service bằng Docker.

    – Áp dụng kiến trúc Microservice rõ ràng.

    – Các service được đóng gói và triển khai bằng Docker.

    – Hệ thống hoạt động ổn định, có dữ liệu minh họa.

    – Có báo cáo đồ án đầy đủ các chương theo quy định.

    Tóm tắt nội dung

    Đề tài tập trung xây dựng một hệ thống học tập trực tuyến đa nền tảng (Web và Mobile) theo kiến trúc Microservice, trong đó các chức năng được tách thành các dịch vụ độc lập và được đóng gói, triển khai bằng Docker.

    Hệ thống cho phép người học tham gia các khóa học lập trình, xem bài giảng, làm bài trắc nghiệm và theo dõi tiến độ học tập. Dữ liệu học tập của người dùng được thu thập và phân tích nhằm phân loại học viên theo năng lực, từ đó đề xuất nội dung học tập phù hợp. Ngoài ra, hệ thống tích hợp chatbot hỗ trợ học tập nhằm hỗ trợ giải đáp và định hướng nội dung học tiếp theo.

    Việc áp dụng Microservice kết hợp Docker giúp hệ thống có khả năng mở rộng, dễ bảo trì, phù hợp với xu hướng phát triển phần mềm hiện đại và đáp ứng yêu cầu về hàm lượng khoa học – công nghệ của đồ án tốt nghiệp ngành Công nghệ thông tin – Ứng dụng phần mềm.

    ➤ Các công nghệ được sử dụng :

    ➤ Kiến Trúc: Microservice

    – Cơ sở dữ liệu: PostgreSQL (Dữ liệu quan hệ), MongoDB (Dữ liệu khóa học/log).

    – Giao tiếp: REST API, API Gateway.

    – Bảo Mật: JWT (JSON Web Token).

    – Container: Docker, Docker Compose.

    ➤ AI: Chatbot API

    ➤ Công Cụ: Git, Postman, Agile/Scrum.

    ➤ DevOps & Hạ tầng: Docker, Docker Compose, Jenkins (CI/CD), AWS (EC2, S3, RDS).

    ➤ Hệ thống tổng quát: được xây dựng theo kiến trúc Microservice, gồm các Microservice chính.

    ➤

    • User Service: Quản lý người dùng & phân quyền.

    • Course Service: Quản lý khóa học, bài học.

    • Quiz Service: Trắc nghiệm & chấm điểm.

    • Analytics Service: Thống kê & cá nhân hóa.

    • AI/Chatbot Service: Phân tích và hổ trợ người học.

    → Mỗi service được đóng gói trong container Docker và triển khai thông qua Docker Compose. Frontend Web và Mobile giao tiếp với các service thông qua API Gateway.
