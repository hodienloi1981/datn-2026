    Tên đề tài 1 Xây dựng Hệ thống Quản lý Bán hàng và Kho cho Doanh nghiệp nhỏ sử dụng Laravel Framework
    Tóm tắt nội dung

    a. Tính cấp thiết của đề tài

    Hiện nay, nhiều cửa hàng và doanh nghiệp nhỏ tại Việt Nam vẫn quản lý bán hàng, kho và doanh thu bằng Excel hoặc ghi chép thủ công, gây:

    - Sai sót dữ liệu

    - Khó quản lý tồn kho

    - Không có báo cáo doanh thu rõ ràng

    Đề tài xây dựng một hệ thống web quản lý bán hàng – kho – khách hàng, phù hợp với thực tế doanh nghiệp nhỏ và vừa.

    b. Backend & Frontend: Laravel Framework

    Laravel là framework PHP được doanh nghiệp tuyển dụng nhiều nhất hiện nay

    - Cung cấp:

    o MVC rõ ràng

    o ORM Eloquent

    o Hệ thống xác thực & phân quyền

    - Giao diện xây dựng bằng Blade Template + Bootstrap / TailwindCSS

    Các yêu cầu cần đạt được:

    a. Các chức năng chính

    a.1. Quản lý người dùng

    - Đăng nhập / đăng xuất

    - Phân quyền:

    o Admin

    o Nhân viên bán hàng

    - Đổi mật khẩu

    a.2. Quản lý sản phẩm

    - Thêm / sửa / xóa sản phẩm

    - Quản lý danh mục sản phẩm

    - Upload hình ảnh sản phẩm

    - Quản lý giá bán

    a.3. Quản lý kho hàng

    - Nhập kho sản phẩm

    - Xuất kho khi bán hàng

    - Tự động cập nhật tồn kho

    - Thông báo sản phẩm sắp hết hàng

    a.4. Quản lý đơn hàng

    - Tạo đơn bán hàng

    - Tính tổng tiền

    - Cập nhật trạng thái đơn hàng

    - Lưu lịch sử giao dịch

    a.5. Quản lý khách hàng

    - Lưu thông tin khách hàng

    - Tra cứu lịch sử mua hàng

    a.6. Báo cáo & thống kê

    - Báo cáo doanh thu theo ngày / tháng

    - Báo cáo sản phẩm bán chạy

    - Báo cáo tồn kho

    - Hiển thị biểu đồ trực quan

    b.1. Backend & Frontend

    - PHP 8.x

    - Laravel 11/12

    - MySQL

    - Blade Template

    - Bootstrap / TailwindCSS

    b.2. Công cụ

    - XAMPP / Laragon

    - Git (quản lý source code)

    Tên đề tài 2 Xây dựng Hệ thống Quản lý Bán hàng và Kho cho Doanh nghiệp vừa và nhỏ (SME) sử dụng Django REST API và React.js
    Tóm tắt nội dung

    a. Bối cảnh & tính thực tiễn

    Hiện nay, phần lớn doanh nghiệp vừa và nhỏ (SME) tại Việt Nam vẫn quản lý bán hàng, kho, đơn hàng bằng Excel hoặc sổ sách, dẫn đến:

    - Dữ liệu rời rạc, dễ sai sót

    - Khó theo dõi tồn kho, doanh thu

    - Không có báo cáo trực quan

    Đề tài hướng đến việc xây dựng một hệ thống web quản lý bán hàng – kho – khách hàng, phù hợp với các cửa hàng, doanh nghiệp nhỏ, đáp ứng nhu cầu thực tế của doanh nghiệp.

    b. Backend: Django & Django REST Framework

    - Xây dựng hệ thống backend bằng Django

    - Thiết kế RESTful API bằng Django REST Framework

    - Quản lý dữ liệu với Django ORM

    - Phân quyền người dùng (Admin – Nhân viên)

    - Bảo mật API bằng JWT Authentication

    - Quản lý dữ liệu sản phẩm, kho, đơn hàng, khách hàng

    c. Frontend: React.js

    - Xây dựng giao diện SPA (Single Page Application)

    - Giao tiếp với backend thông qua REST API

    - Quản lý trạng thái bằng Context API hoặc Redux

    - Thiết kế giao diện hiện đại, dễ sử dụng, phù hợp môi trường doanh nghiệp
    Các yêu cầu cần đạt được:

    a. Các chức năng chính của hệ thống

    a.1. Quản lý người dùng

    - Đăng nhập / đăng xuất hệ thống

    - Phân quyền:

    o Admin: toàn quyền quản lý

    o Nhân viên: tạo đơn hàng, xem sản phẩm

    - Đổi mật khẩu, bảo mật tài khoản

    a.2. Quản lý sản phẩm

    - Thêm, sửa, xóa sản phẩm

    - Quản lý danh mục sản phẩm

    - Quản lý giá bán, hình ảnh sản phẩm

    - Theo dõi số lượng tồn kho

    a.3. Quản lý kho hàng

    - Nhập kho, xuất kho

    - Cập nhật tồn kho tự động theo đơn hàng

    - Cảnh báo sản phẩm sắp hết hàng

    a.4. Quản lý đơn hàng

    - Tạo đơn bán hàng

    - Cập nhật trạng thái đơn hàng (Chờ xử lý – Đã hoàn thành – Đã hủy)

    - Tính tổng tiền đơn hàng

    - Lưu lịch sử giao dịch

    a.5. Quản lý khách hàng

    - Lưu thông tin khách hàng

    - Tra cứu lịch sử mua hàng

    - Hỗ trợ chăm sóc khách hàng

    a.6. Báo cáo & thống kê

    - Thống kê doanh thu theo ngày / tháng

    - Thống kê sản phẩm bán chạy

    - Báo cáo tồn kho

    - Biểu đồ trực quan (Chart.js hoặc Recharts)

    b. Công nghệ & thư viện sử dụng

    b.1.Backend

    - Django

    - Django REST Framework

    - Django ORM

    - Simple JWT (Xác thực)

    - PostgreSQL hoặc MySQL

    b.2.Frontend

    - React.js

    - React Router

    - Axios

    - Context API / Redux

    - TailwindCSS hoặc Material-UI

    - Chart.js / Recharts
