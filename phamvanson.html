    PHẠM VĂN SƠN

    Tên đề tài 1 Kiểm thử website (tin tức, bán hàng, giáo dục…)
    Tóm tắt nội dung

    - Dựa vào sản phẩm là các website, sinh viên trao đổi với khách hàng hoặc đối tác về những nội dung sẽ test (theo hợp đồng hoặc thoả thuận giữa các bên) từ đó phân tích và xác định nội dung test.

    - Lập kế hoạch kiểm thử (dạng văn bản và file excel theo mẫu GVHD cung cấp)

    - Thiết kế test case (theo mẫu GVHD cung cấp)

    - Nghiên cứu và lựa chọn công cụ kiểm thử phù hợp với website cần test (ví dụ: Katalon Studio, Test Complete, PostMan, Selenium IDE, Nunit, Junit…)

    - Thực hiện test bằng tay (manual test) hoặc test tự động dùng tool (automation test)
    Các yêu cầu cần đạt được:

    4.1. Tìm hiểu đề tài, xác định mục đích, yêu cầu, nội dung nghiên cứu và cấu trúc một báo cáo (KLTN/TTTN)

    4.2. Hệ thống hoá và trình bày những lý thuyết liên quan đến đề tài KLTN/TTTN

    4.3. Dựa vào sản phẩm kiểm thử: phân tích yêu cầu khách hàng và trên thực tế sản phẩm, lập kế hoạch kiểm thử chi tiết (dạng file word/excel)

    4.4. Thiết kế test case chi tiết có dữ liệu kiểm thử (test data) để thực hiện test bằng tay hoặc bằng tool

    4.5. Từ test data thực hiện test và viết báo cáo kiểm thử (báo cáo lỗi) xuất ra từ phần mềm dưới dạng file PDF/SCV hoặc theo mẫu sinh viên đã phân tích và hoàn thành KLTN/TTTN

    4.6. Đánh giá kết quả đạt được và những hạn chế (nếu có)
